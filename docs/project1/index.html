비개발자를 위한 개발 용어 사전

개발자와 소통하거나 IT 프로젝트에 참여할 때 꼭 알아야 할
기본 개발 용어들을 쉬운 비유와 함께 알아봅시다.

키보드 방향키 또는 화면의 버튼으로 시작하세요.

코드(Code)와 프로그래밍(Programming)

컴퓨터가 이해할 수 있는 언어로 작성된 명령어 집합이 바로 '코드'입니다.

코드 (Code)

요리 레시피처럼 컴퓨터에게 "이렇게 해라"라고 알려주는 글입니다. 예: "사용자가 버튼을 누르면 팝업창을 띄워라"

프로그래밍 (Programming)

코드를 작성하는 행위입니다. 마치 요리사가 레시피를 만드는 것처럼 개발자가 코드를 만드는 과정입니다.

💡 "코드를 짠다" = "프로그램을 만든다"는 뜻입니다.

프로그래밍 언어

사람이 영어, 한국어로 대화하듯 컴퓨터와 대화하기 위한 언어입니다.
각각 특징과 용도가 다릅니다.

JavaScript 웹사이트를 움직이게 하는 언어 (버튼 클릭, 애니메이션 등)
Python 데이터 분석, AI 개발에 많이 쓰이는 쉬운 언어
Java 대기업 시스템, 안드로이드 앱 개발에 주로 사용

💡 각 언어마다 문법이 조금씩 다르지만, 기본 개념은 비슷합니다.

프론트엔드 vs 백엔드

웹사이트나 앱은 크게 '보이는 부분'과 '보이지 않는 부분'으로 나뉩니다.

프론트엔드

사용자가 직접 보고 만지는 부분

• 버튼, 메뉴, 화면 디자인

• 클릭했을 때 반응

• 카페 매장의 인테리어와 메뉴판

백엔드

사용자가 보지 못하는 뒤쪽 처리

• 데이터 저장, 계산, 처리

• 로그인, 결제 처리

• 카페 주방과 재료 창고

💡 "풀스택 개발자" = 프론트엔드와 백엔드를 모두 할 수 있는 개발자

데이터베이스(DB)와 서버

정보를 저장하고 처리하는 컴퓨터 시스템들입니다.

사용자
(클라이언트)

스마트폰, 컴퓨터

서버

요청을 처리하는 컴퓨터

데이터베이스

정보 저장소

서버: 24시간 켜져있는 강력한 컴퓨터. 은행의 ATM기계처럼 요청을 받아 처리
데이터베이스: 엑셀처럼 정보를 체계적으로 저장하는 거대한 파일 시스템

API (에이피아이)

Application Programming Interface - 서로 다른 프로그램이 대화하는 방법입니다.

API란?

카페에서 주문받는 직원처럼, 요청을 받아 적절한 응답을 돌려주는 연결고리입니다.

실제 예시

"날씨 API"에 "서울 날씨"를 요청하면 "맑음, 25도"라는 정보를 받습니다.

API 요청 예시:

요청: "오늘 서울 날씨 알려줘"
응답: { "지역": "서울", "날씨": "맑음", "온도": "25도" }

💡 카카오톡 로그인, 결제 시스템도 모두 API를 통해 작동합니다.

개발 환경과 도구들

개발자들이 사용하는 주요 도구들을 알아봅시다.

IDE (통합개발환경)

코드를 작성하는 프로그램. 워드프로세서의 개발자 버전

예: VS Code, IntelliJ

Git/GitHub

코드의 변경사항을 기록하고 백업하는 시스템

구글 드라이브의 개발자 버전

터미널/CLI

명령어로 컴퓨터를 조작하는 검은 화면

DOS 시절의 까만 화면

라이브러리/패키지

미리 만들어진 코드 묶음. 레고 블록 같은 것

npm, pip로 설치

웹 개발 기초 용어

웹사이트를 만들 때 사용하는 핵심 기술들입니다.

HTML 웹페이지의 골격. 제목, 문단, 이미지 등의 구조를 만듦

집을 지을 때 기둥과 벽을 세우는 것

CSS 웹페이지의 디자인. 색깔, 크기, 위치 등을 꾸며줌

집에 페인트칠하고 가구를 배치하는 것

JavaScript 웹페이지를 움직이게 함. 클릭하면 반응하는 모든 기능

집에 전등 스위치와 자동문을 설치하는 것

💡 HTML + CSS + JavaScript = 완전한 웹페이지

개발 프로세스 용어

개발 프로젝트에서 자주 나오는 용어들입니다.

버그(Bug) / 디버깅(Debugging)

프로그램의 오류. 의도와 다르게 작동하는 것을 고치는 작업

예: 버튼을 눌렀는데 아무 반응이 없음

배포(Deploy)

완성된 프로그램을 실제 사용자들이 쓸 수 있도록 인터넷에 올리는 것

책을 써서 서점에 진열하는 것과 비슷

테스트(Test)

프로그램이 제대로 작동하는지 확인하는 과정

요리를 하고 간을 맞춰보는 것

버전 관리

코드의 변경 사항을 추적하고 관리하는 시스템

문서의 "변경 내용 추적" 기능과 비슷

자주 나오는 IT 용어

회사에서 개발자와 대화할 때 알아두면 좋은 용어들입니다.

🔄 프레임워크 vs 라이브러리

프레임워크는 집의 뼈대, 라이브러리는 가구

☁️ 클라우드

인터넷으로 접근하는 컴퓨터 서비스 (AWS, Azure)

🏗️ 아키텍처

시스템의 전체적인 구조와 설계

📱 반응형 디자인

PC, 모바일에서 모두 잘 보이는 웹페이지

🔐 보안 (Security)

해킹 방지, 개인정보 보호 등

⚡ 성능 최적화

프로그램을 더 빠르고 효율적으로 만드는 것

🎨 UI/UX

사용자 인터페이스 / 사용자 경험

🚀 애자일/스프린트

빠른 주기로 개발하는 방법론

AI, 머신러닝, 딥러닝 용어

요즘 가장 핫한 AI 기술 관련 용어들을 쉬운 비유로 알아봅시다.

🤖 인공지능 (AI)

사람처럼 생각하고 판단하는 컴퓨터 프로그램

예: 시리, 알렉사, ChatGPT

🧠 머신러닝 (ML)

컴퓨터가 데이터를 보고 스스로 학습하는 방법

아이가 경험을 통해 배우는 것처럼

🔗 딥러닝 (DL)

뇌의 뉴런을 모방한 복잡한 머신러닝

사람 뇌처럼 여러 층으로 정보 처리

📋 알고리즘

문제를 해결하는 단계별 방법

요리 레시피처럼 순서가 있는 규칙

🎯 모델

학습이 완료된 AI의 "뇌"

숙련된 의사의 진단 능력과 비슷

📚 데이터셋

AI 학습용 정보 모음

학생의 교과서와 문제집

🎓 훈련/학습

AI가 데이터를 보고 패턴을 익히는 과정

학생이 문제집을 반복해서 푸는 것

🔮 예측

학습한 내용으로 새로운 상황에 답하기

경험 많은 기상캐스터의 날씨 예보

💡 AI > 머신러닝 > 딥러닝 순으로 포함 관계입니다.
